# 智能网联汽车新闻自动化监测方案

## 一、方案概述

利用 TRAE 的定时任务（Schedule）+ 内置浏览器 MCP 工具 + AI 智能分析能力，搭建一个全自动的智能网联汽车新闻监测系统：

- **每日定时采集**：每天上午自动抓取各大信息源的最新新闻，生成当日新闻简报（HTML）
- **月底自动汇总**：每月最后一天自动汇总全月数据，生成月度分析报告（HTML）
- **全自动化运行**：无需人工干预，定时触发、自动执行、自动输出

## 四、技术架构

### 4.1 整体流程

```
Schedule 定时触发
    │
    ▼
AI Agent 解读 Message 模板
    │
    ▼
浏览器 MCP 抓取各网站新闻列表
    │
    ▼
逐条提取正文 + 关键词分类 + AI 摘要
    │
    ▼
与历史数据去重对比
    │
    ▼
生成 HTML 报告 + 更新数据索引
```

### 4.2 目录结构

```
e:\trae_solo\自动化监测智能网联汽车新闻\
  ├── reports\
  │     ├── daily\              # 每日新闻简报
  │     │     ├── 2026-07-06.html
  │     │     └── ...
  │     └── monthly\            # 月度分析报告
  │           ├── 2026-07.html
  │           └── ...
  └── data\
        ├── news_index.json     # 全量新闻索引（用于去重和月度汇总）

### 4.3 数据持久化策略

- 使用 `news_index.json` 存储所有已抓取新闻的元数据（URL、标题、日期、分类、来源）
- 每次执行时读取已有索引，对比 URL 去重，仅处理新增新闻
- 月度报告汇总时从该索引读取当月全部数据

## 五、自动化任务设计

### 5.1 每日新闻监测任务

- **名称**：智能网联汽车每日新闻监测
- **Cron**：`0 9 * * 1-5`（工作日每天上午 9:00 执行）
- **时区**：Asia/Shanghai
- **执行流程**：
  1. 使用浏览器 MCP 依次导航到各信息源页面
  2. 获取页面快照，分析列表结构
  3. 提取**昨日 + 今日**（任务执行当天）发布或更新的新闻条目（标题、日期、链接）
  4. 逐条进入详情页提取正文摘要
  5. 通过 WebSearch 补充搜索关键词相关新闻（**限定时间范围为昨日 + 今日**）
  6. 基于关键词库自动分类为五大类别
  7. 读取 `news_index.json` 进行 URL 去重
  8. AI 为每条新闻生成一句话摘要
  9. 生成当日 HTML 简报，按分类组织
  10. 将新增新闻追加到 `news_index.json`
- **输出**：`reports/daily/YYYY-MM-DD.html`

### 5.2 月度汇总报告任务

- **名称**：智能网联汽车月度分析报告
- **Cron**：`0 10 28-31 * *`（每月末最后几天尝试执行，Message 中包含判断逻辑：仅当为当月最后一天时才生成报告）
  - 备选方案：每月 1 日上午生成上个月的报告（`0 10 1 * *`）
- **时区**：Asia/Shanghai
- **执行流程**：
  1. 从 `news_index.json` 读取当月全部新闻数据
  2. 统计各类别新闻数量、来源分布
  3. AI 分析本月热点趋势、重要政策节点
  4. 生成 Top 10 重要新闻深度摘要
  5. 按五大类别组织月度汇编
  6. 生成 HTML 月度报告（含数据统计、趋势分析）
- **输出**：`reports/monthly/YYYY-MM.html`

### 5.3 Message 模板关键设计要点

每日任务的 Message 模板需要明确包含：

1. **目标网站列表**（含完整 URL）
2. **关键词分类规则**（五大分类 + 对应关键词）
3. **输出路径规范**（固定到 `reports/daily/`）
4. **数据索引路径**（固定到 `data/news_index.json`）
5. **去重逻辑**（对比已有索引的 URL）
6. **HTML 报告格式要求**（标题、分类、摘要、来源、日期、原文链接）
7. **异常处理**（某个网站无法访问时跳过，不影响其他网站采集）

## 六、HTML 报告格式设计

> **设计原则**：使用 `frontend-design` skill 的设计理念，生成**高质量、有设计感的 HTML 报告**，避免千篇一律的模板化外观。每份报告应具备独特的视觉风格，体现专业性和可读性。

### 6.0 视觉设计规范

**设计方向**：采用「高端行业报告」风格，兼具专业感与现代审美。

- **Typography（字体）**：
  - 标题/数据展示：选择有辨识度的字体（如 Noto Serif SC、思源宋体）
  - 正文/摘要：清晰易读的正文字体（如 Noto Sans CJK SC）
  - 避免使用 Arial、Inter 等通用字体

- **Color & Theme（配色）**：
  - 以深蓝色系为主色调，搭配锐利的强调色（如橙红色或金黄色）
  - 使用 CSS 变量保持全局一致性
  - 日报采用浅色背景（专业简报风格），月报可采用深色主题（数据报告风格）

- **Motion & Interaction（动效）**：
  - 页面加载时使用 staggered reveal 动画（CSS animation-delay）
  - 新闻条目 hover 时有微交互反馈
  - 分类标签使用色彩编码，一目了然

- **Layout（布局）**：
  - 日报：简洁卡片式布局，分类区块清晰分隔
  - 月报：仪表盘式布局，含数据统计图和趋势图表

- **数据可视化**：使用纯 CSS/SVG/JS 实现统计图表（饼图、柱状图、时间线），不依赖第三方图表库

### 6.1 每日简报结构

每条新闻直接展示**标题 + AI 生成的一段话摘要**（一段连贯的文字，概括核心事件、关键数据、影响与意义），用户无需点击展开即可快速浏览全部新闻。如需查看原文详情，点击标题链接即可跳转。

```

**月度报告与日报的区别**：
- 月度报告额外包含数据概览统计图表和趋势分析
- "十大重要新闻"板块的摘要为更长的段落（深度摘要 vs 普通摘要）
- 分类汇编中的每条新闻保持与日报相同的格式（标题 + 一段话摘要）

## 七、实施步骤

### 步骤 1：创建目录结构
在工作文件夹下创建 `reports/daily/`、`reports/monthly/`、`data/` 三个子目录。

### 步骤 2：初始化数据文件
创建 `data/news_index.json`（空数组 `[]`）和 `data/last_update.json`（初始时间戳）。

### 步骤 3：手动验证信息源
逐一访问各目标网站，确认页面结构和可抓取性，记录各网站的新闻列表 CSS 选择器。

### 步骤 4：创建每日报告 HTML 模板
设计一个美观的每日新闻简报 HTML 模板，包含样式、分类区块、新闻条目格式。

### 步骤 5：创建月度报告 HTML 模板
设计月度分析报告模板，包含统计图表、分类汇编、趋势分析等区块。

### 步骤 6：配置每日定时任务
通过 TRAE Schedule 创建每日任务，编写完整的 Message 模板。

### 步骤 7：首次手动执行测试
手动触发一次每日任务，验证抓取效果、分类准确性、报告质量。

### 步骤 8：配置月度定时任务
基于每日任务验证结果，创建月度汇总任务。

### 步骤 9：调优迭代
根据执行结果优化关键词库、报告格式、信息源列表。

## 八、注意事项

1. **政府网站结构变化**：工信部等网站可能不定期改版，Message 模板中需包含页面结构检测的容错逻辑
2. **网站访问限制**：部分政府网站可能有反爬措施，需要合理设置请求间隔
3. **数据准确性**：AI 分类可能存在误差，可在月度报告生成时进行二次校验
4. **跨会话数据衔接**：每次定时任务触发是独立会话，必须通过 `news_index.json` 文件实现数据持久化和去重
5. **月度任务触发时机**：由于 Cron 不支持"每月最后一天"的精确表达，建议用每月 1 日生成上月报告的方案
6. **信息源动态调整**：可根据实际效果定期增减监测的信息源

## 九、关键决策记录

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 输出格式 | HTML | 可视化效果好，支持交互，浏览器直接查看 |
| 采集方式 | 浏览器 MCP + WebSearch | 政府网站多为动态渲染，需浏览器工具；WebSearch 补充遗漏 |
| 数据存储 | JSON 文件 | 轻量级，无需数据库，适合 TRAE 环境 |
| 去重策略 | URL 对比 | 简单可靠，新闻 URL 通常具有唯一性 |
| 每日执行时间 | 工作日 9:00 | 政府网站通常在工作日更新内容 |
| 月度执行时间 | 每月 1 日 10:00 | 生成上月完整报告，避免月末遗漏 |
